## OVERVIEW
TypeScript ESM plugin for OpenCode. Discovers 9router models at startup and injects provider/model config without hardcoding model lists in `opencode.json`.

```text

├── src/                 # plugin, CLI, model mapping, cache
```

## CONVENTIONS
- Package is ESM: `"type": "module"`; source imports local modules with `.js` suffix.
- TypeScript config: `target` ES2022, `module` NodeNext, `moduleResolution` NodeNext, `declaration: true`, `strict: true`, `skipLibCheck: true`, `types: ["node"]`.
- Peer dependency: `@opencode-ai/plugin >=1.14.0`; dev dependency currently `^1.14.48`.
- Runtime plugin reads env: `OPENCODE_9ROUTER_URL`, `OPENCODE_9ROUTER_API_KEY`, `OPENCODE_9ROUTER_TIMEOUT_MS`.
- CLI uses Node stdlib only for config edits: `fs/promises`, `path`, `os`, `readline/promises`, `child_process`.
- Config writes create backups for existing files and write atomically through temp file + rename.
- Prefer Bun locally: `bun run build`. `package.json` still has npm-oriented publish lifecycle.

## ANTI-PATTERNS (THIS PROJECT)
- Do not hardcode 9router model lists in `opencode.json`; runtime discovery owns model list.
- Do not edit commented `.jsonc` through fallback CLI; it intentionally refuses comment-preservation risk.
- Do not add config fields that violate `https://opencode.ai/config.json`; OpenCode hard-fails invalid config.
- Do not write API keys to docs, config examples, logs, or generated AGENTS.md.
- Do not treat `dist/` as source; it is generated and ignored.

## UNIQUE STYLES
- Plugin entry is small; CLI carries most setup/troubleshooting behavior.
- Model metadata enriches discovered IDs from `models.dev`, but falls back to a permissive template.
- `models.dev` payload cache lives outside repo in `~/.cache/opencode-9router-plus/models.dev.json`.
- Both `bun.lock` and `package-lock.json` are present; avoid dependency edits unless lockfile policy is explicit.
